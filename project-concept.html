Project Concept

Summary

Our project is to continue building on Grd.me's existing source code to expand its functionalities and make it a more viable product for any kind of user. Grd.me is an open source text encryption broswer extension for Chrome that enables users to send private text on any messaging platform.



Our goals are to make Grd.me a multi-device extension to improve its UI experience so that users with either some or no prior knowledge of cryptography can use Grd.me easily and with minimal instruction, and to possibly use watermarks in profile pictures to associate keys with users.



Currently, users are asked to create both a public a private key, and then must share their public key with others in order to send encrypted messages to each other. Ideally, we'd like to hide these processes from the user to make the tool as streamlined as possible. We will accomplish this by improving the UI of the current tool and include detailed, step-by-step instructions for users to make the process easier to follow.



Upon completion of this goal, we will also work to make the service functional on multiple devices so that users will have the option to send encrypted messages on different devices and have cross-device support. This will make it possible, for example, for users to send an encrypted message to a friend through Facebook on their desktop, and also see it on their phone.



Our stretch goal is to hide public encryption keys in pictures, focusing on .png files, using stenography. Hiding keys in pictures would allow users to display their public keys in profile pictures on various websites (i.e. twitter and github), which would offer other users a decentralized way of verifying key ownership.
